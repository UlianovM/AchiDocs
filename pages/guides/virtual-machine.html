Виртуальная машина

Установка

ПакетЗачем нужен
qemu-full Сам эмулятор/гипервизор, полная версия со всеми архитектурами и бэкендами
libvirt Демон-обёртка над QEMU — единый API для управления ВМ, сетями, хранилищами
virt-manager GUI для управления ВМ (использует GTK, работает на Wayland через XWayland)
edk2-ovmf UEFI-прошивка для виртуальных машин (вместо legacy BIOS)
dnsmasq DNS/DHCP-сервер для виртуальной NAT-сети
vde2 Виртуальный Ethernet — продвинутая сеть между ВМ
openbsd-netcatНужен libvirt для некоторых сетевых операций
bridge-utils Создание сетевых мостов (bridged networking)

Установка пакетов из основных репозиториев:

$
sudo pacman -S qemu-full edk2-ovmf \ libvirt virt-manager \ dnsmasq openbsd-netcat \ vde2

Установка пакета bridge-utils из AUR:

$
git clone https://aur.archlinux.org/bridge-utils.git \ makepkg -siCD --skippgpcheck bridge-utils \ rm -rf bridge-utils

Настройка

Настройка сервисов и прав

$
sudo systemctl enable --now libvirtd.service
$
sudo usermod -aG libvirt,kvm username
$
sudo nano /etc/libvirt/libvirtd.conf
/etc/libvirt/libvirtd.conf
conf
85
108
unix_sock_group = "libvirt" unix_sock_rw_perms = "0770"
$
sudo systemctl restart libvirtd

Включение виртуальной сети

$
sudo virsh net-start default
$
sudo virsh net-autostart default

Использование

Запуск менеджера виртуальных машин:

$
virt-manager
